# KVCDN Domain Language

This file names the core concepts of the kvcdn system. Use these terms in architecture discussions and module names.

- **Artifact**: a stored KV-cache file plus its metadata (name, SHA-256, size, dtype, token/layer counts, visibility). The unit of upload, download, list, and delete.
- **Customer**: the authenticated tenant. Derived deterministically from either an OIDC `sub` claim (`oidc:<sub>`) or an org-scoped API key (`org:<org>`). Forms the top-level storage namespace.
- **Org / Project**: human-readable namespace scopes inside a Customer. A route contains `/:org/:project` and storage keys include both.
- **Object Store**: the external blob storage system (currently S3-compatible) that holds Artifact data and metadata sidecars.
- **Metadata Sidecar**: a `.meta.json` JSON file stored next to an Artifact's data object, containing the Artifact record.
- **Presigned URL**: a time-limited, direct-upload/download URL generated by the backend and returned to the CLI.
- **API Key**: a deterministic, org-scoped bearer token derived from `KVCDN_API_KEY_SEED` + org slug via HKDF-SHA256.
- **Visibility**: access level of an Artifact — `private` (only the owning Customer) or `public` (readable without authentication).
- **OIDC Provider**: the external OpenID Connect issuer (Pocket ID) that authenticates CLI users.
- **Backend**: the Fly.io-hosted Fastify service that issues Presigned URLs, validates tokens, and writes Metadata Sidecars.
- **Transfer Adapter**: the CLI-side module that moves Artifact bytes to/from a Presigned URL, independent of the REST API client.
- **Credential Store**: the CLI-side module that persists OIDC tokens, encrypted at rest under a key from the OS keyring or a fallback file.
