# Local Kubernetes Deployment

This directory contains Helm values and Kubernetes manifests for running the kvcdn stack locally in a [`kind`](https://kind.sigs.k8s.io/) cluster.

## Components

| Component | Namespace | Purpose |
|---|---|---|
| kvcdn API | `kvcdn` | TypeScript/Fastify backend that serves OIDC discovery, validates JWTs, and issues MinIO presigned PUT URLs. |
| MinIO | `storage` | S3-compatible object store for uploaded artifacts. |
| Local registry | (host) | Docker registry on `localhost:5001` so images built on the host can be pulled by cluster pods. |

## Architecture

```
┌─────────────────┐     port-forward      ┌─────────────────────────────┐
│   kvcdn CLI     │ ────────────────────▶ │  kvcdn API (kind: kvcdn)   │
│  (host binary)  │   http://localhost:8080  │  Fastify + mock OIDC        │
└─────────────────┘                       └──────────────┬──────────────┘
                                                       │
                                                       │ presigned PUT
                                                       ▼
                                              ┌─────────────────┐
                                              │ MinIO (kind:    │
                                              │  storage)       │
                                              │  :9000          │
                                              └─────────────────┘
```

The CLI uses the same environment-variable contract as production:

- `KVCDN_API_URL`
- `KVCDN_ISSUER_URL`
- `KVCDN_CLIENT_ID`

For the local stack these default to:

```bash
export KVCDN_API_URL=http://localhost:8080
export KVCDN_ISSUER_URL=http://localhost:8080/oidc
export KVCDN_CLIENT_ID=kvcdn-cli
```

## Quick start

1. Make sure you have installed:
   - `kind`
   - `kubectl`
   - `helm`
   - `docker` or `podman`
   - `cargo` and Rust toolchain
   - `jq` and `xmllint` (`libxml2-utils` on Debian/Ubuntu)

2. From the repo root run:

   ```bash
   cargo test
   cd backend && npm test && cd ..
   ```

## Manual steps

To run the local stack manually:

```bash
# 1. Cluster + registry
./scripts/kind-with-registry.sh

# 2. Build and push images
cargo build --release
docker build -t localhost:5001/kvcdn-api:local backend/
docker push localhost:5001/kvcdn-api:local
docker build -t localhost:5001/kvcdn-cli:local .
docker push localhost:5001/kvcdn-cli:local

# 3. Deploy MinIO
kubectl apply -f deploy/local/namespace/storage.yaml
helm repo add minio https://charts.min.io
helm install minio minio/minio -n storage -f deploy/local/minio/minio-values.yaml --wait

# 4. Deploy kvcdn API
kubectl apply -f deploy/local/namespace/kvcdn.yaml
kubectl apply -f deploy/local/backend/

# 5. Port-forward services
kubectl -n kvcdn port-forward svc/kvcdn-api 8080:3000 &
kubectl -n storage port-forward svc/minio 9000:9000 9001:9001 &

# 6. Log in and upload
export KVCDN_API_URL=http://localhost:8080
export KVCDN_ISSUER_URL=http://localhost:8080/oidc
export KVCDN_CLIENT_ID=kvcdn-cli
./target/release/kvcdn login --no-browser
# open the printed URL in a browser to complete the OIDC callback.
./target/release/kvcdn upload ./path/to/model.kv --name my-model --project default
```

```bash
./scripts/kind-with-registry.sh --teardown
```

## Rootless container runtimes

If you are running `kind` with rootless podman or rootless Docker, the cluster nodes need cgroup delegation. The setup script attempts to use `systemd-run --user --scope --property=Delegate=yes` automatically. If that fails with an error such as:

```text
running kind with rootless provider requires setting systemd property "Delegate=yes"
```

apply the persistent fix on the host as root and reboot or restart `user@$UID.service`:

```bash
sudo mkdir -p /etc/systemd/system/user@.service.d/
sudo tee /etc/systemd/system/user@.service.d/delegate.conf <<'EOF'
[Service]
Delegate=yes
EOF
sudo systemctl daemon-reload
```

Then restart your user manager:

```bash
systemctl --user daemon-reexec
```

## Troubleshooting

### `ImagePullBackOff` for `localhost:5001/...`

1. Confirm the registry is running: `curl http://localhost:5001/v2/_catalog`.
2. Confirm the image was pushed to `localhost:5001/<name>:local`.
3. Confirm the `kind` nodes can resolve `localhost:5001` via the `hosts.toml` generated by `scripts/kind-with-registry.sh`:
   ```bash
   docker exec kvcdn-local-control-plane cat /etc/containerd/certs.d/localhost:5001/hosts.toml
   ```

### OIDC discovery fails

Check that `KVCDN_ISSUER_URL` in `deploy/local/backend/configmap.yaml` matches the host:port you use for `kubectl port-forward`. If you change the local port, update both the configmap and your shell exports, then restart the backend deployment.

### MinIO bucket does not exist

The `minio-values.yaml` creates an `artifacts` bucket at install time. If the bucket is missing, port-forward MinIO and use the MinIO Console (`http://localhost:9001`, credentials `minioadmin/minioadmin`) or `mc` to create it.

### Login callback never completes

The CLI prints an authorization URL and waits for a `GET` to its local callback server. With `--no-browser` you must visit the URL in a browser. Make sure nothing else is bound to the callback port printed in the URL.
